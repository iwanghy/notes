# SDK_RTOS迁移

- VISION_APPS
  - 目录变化
    - 路径更合理，可读性更好
    - 迁移有成本
  - 内存分区
    - 完全不一样
    - 需要基于新SDK进行基本功能的验证
    - 百度开发的代码中写死地址的需要调试
  - tidl示例变化不大
    - 好消息，A核代码不需要大改
  - TI RTOS->free RTOS
    - 我们几乎不使用RTOS原生接口，工作量不大
- TIOVX
  - 头文件变化
    - A核代码需要进行修改
  - codec(MCU2_1->A核)
    - 需要基于v4l2接口重新适配
- TIDL
  - 工具有更新，支持了BatchNum参数
    - 必然伴随重新转模型
  - 主要算法都支持了BatchNum参数
    - 我们增加的Batch的patch特性可能不再需要，重新评估
  - 可能更好的支持batch特性，能达到更好的并行性能
    - 需要评估
